Обозначают 5 принципов Объектно-ориентированного программирования. Помогают создавать понятный, гибкий и легко поддерживаемый код. Они помогают решать типичные проблемы объектно-ориентированных программ: сильно связанные классы, трудности с тестированием, проблемы с расширяемостью, неустойчивость к изменениям

### Принцип единственной ответственности (SRP)
 у каждого класса должна быть только одна причина для изменения, то есть класс должен отвечать только за одну задачу
 **Пример**: вместо того чтобы создавать класс, который обрабатывает и логику, и отображение данных, лучше разделить их на два отдельных класса: один для логики, другой для отображения.

### Открытости/закрытости (OCP)
программные сущности должны быть открыты для расширения, но закрыты для изменения. Это значит, что функциональность системы можно расширять, не изменяя уже существующий код.
**Пример**: вместо изменения существующего класса для добавления новой функции лучше создать новый класс, который расширяет функциональность исходного класса.

### Подстановки Лисков (LSP)
объекты подклассов должны быть заменяемы объектами их суперклассов без нарушения корректности программы. Это означает, что наследуемый класс должен дополнять, а не изменять поведение базового класса.
**Пример**: если есть класс «Птица» с методом «летать», то подклассы «Воробей» и «Пингвин» должны корректно наследовать этот метод. Если «Пингвин» не может летать, нужно изменить иерархию классов, чтобы избежать нарушения LSP.

### Разделения интерфейса (ISP)
клиенты не должны зависеть от методов, которые они не используют. Вместо этого интерфейсы должны быть разделены на более мелкие, специализированные интерфейсы, чтобы клиенты могли реализовывать только те методы, которые им нужны.
**Пример**: вместо создания общих интерфейсов следует создавать специфические интерфейсы для конкретных клиентов

### Инверсия зависимостей (DIP)
Классы должны зависеть от абстракций, а не от конкретных реализаций. Высокоуровневые модули не должны зависеть от низкоуровневых модулей, оба типа модулей должны зависеть от абстракций.
